# 山淮筑设计规范

本文档沉淀当前项目登录页和首页/后台入口的视觉与交互规范，供后续页面、组件和功能迭代时统一遵循。

## 设计基准

- 主要来源：
  - `ui/src/features/auth/components/LoginPage.tsx`
  - `ui/src/features/auth/components/LoginForm.tsx`
  - `ui/src/routes/app/admin.tsx`
  - `ui/src/components/layout/AppSidebar.tsx`
  - `ui/src/features/projects/components/ProjectsPage.tsx`
  - `ui/src/features/projects/components/ProjectDetailPage.tsx`
  - `ui/src/index.css`
- 产品定位：建设项目现场管理后台，服务项目、班组、工人、考勤、权限与通知等高频管理场景。
- 第一原则：界面要克制、专业、可扫描，优先支持长期办公使用，而不是营销型展示。
- 当前公开落地页仍有 Quax/Miku 模板痕迹，不能作为后续业务页面的风格基准。

## 品牌与语气

- 产品名称统一使用「山淮筑」。
- 后台副标题优先使用「管理后台」「项目、班组、工人、考勤统一管理」等业务化描述。
- 品牌标识当前使用绿色方形底加「山」字，适用于登录页、侧栏和后台顶部。
- 文案要简洁、具体、面向工程现场管理：
  - 推荐：`项目管理`、`项目台账`、`新增项目`、`考勤待补图`、`统一查看项目、单位、班组、人员与考勤基础信息。`
  - 避免：`Quick Start`、`Open Source Starter Template`、`Enter App`、`Star on GitHub`、`Miku`、通用模板宣传语。
- 操作文案使用动词 + 对象：`新增项目`、`编辑项目`、`导出档案`、`返回项目列表`、`查看全部通知`。

## 色彩系统

全局 token 以 `ui/src/index.css` 为准。

| 用途 | 亮色 | 暗色 | 使用场景 |
| --- | --- | --- | --- |
| 主色 primary | `#0f6b5d` | `#1aa38e` | 主按钮、激活态、链接 hover、侧栏品牌 |
| 主色 hover | `#0b5148` | 按语义加深 | 主按钮 hover |
| 登录主色 | `#0f7d6f` / `#0f9b88` | `#28c7b4` | 登录页品牌、表单 focus、登录按钮 |
| 次级底色 | `#ecf4f1` | `#12332f` | accent、secondary、侧栏 active 背景 |
| 页面浅底 | `#f8faf9` | `muted/30` | 筛选条、表头、统计小卡 |
| 文本主色 | slate-950 | foreground / white | 标题和主要内容 |
| 弱文本 | slate-500/600 | muted-foreground / white alpha | 辅助说明、表格次信息 |
| 预警 amber | amber-50/200/700 | amber-950/900/300 | 待关注、停工、通知点 |
| 危险 red | red-50/200/700 | red-950/900/300 | 删除、异常、预警 |
| 成功 emerald | emerald-50/200/700 | emerald-950/900/300 | 正常、在建、有效 |

禁止继续扩散旧模板的青紫/粉紫渐变主视觉，例如 `miku-primary` 到 `miku-accent` 的大面积渐变按钮或标题。

## 字体与排版

- 全局字体：`Inter, system-ui, sans-serif`。
- 字间距保持正常，使用 `tracking-normal` 或默认值，不使用负字距。
- 标题层级：
  - 登录页品牌大标题：42px，xl 可到 48px，`font-semibold`，行高约 1.22。
  - 登录表单标题：28px，`font-semibold`。
  - 后台页面标题：24px 左右，`font-semibold`，不使用过重的营销标题。
  - 卡片/区块标题：16px，`font-semibold`。
  - 表格、筛选、辅助信息：12px 到 14px。
- 后台页面应控制信息密度，更多使用紧凑标题、说明、表格和过滤器，避免大面积 hero。

## 登录页规范

登录页是当前最完整的品牌化页面。

- 布局：
  - 使用 full-bleed 工地图片背景：`ui/public/login-construction-bg.jpg`。
  - 叠加浅色/暗色横向渐变遮罩，保证文字和表单可读。
  - 内容容器 `max-w-7xl`，左右 padding 20px 起，桌面端使用左侧价值说明 + 右侧登录卡片。
  - 移动端隐藏左侧大宣传区，保留品牌头部和登录表单。
- 品牌头部：
  - 左侧使用 56px 方形「山」字标，圆角 `rounded-lg`，主色背景。
  - 标题为「山淮筑」，副标题为「管理后台」。
- 左侧说明：
  - 胶囊标签：`现场管理专用入口`。
  - 价值点保持三项以内：安全合规、精细管控、协同高效。
  - 图标使用 lucide，56px 圆形淡绿色容器。
- 登录卡片：
  - 使用玻璃拟态白色/暗色面板，`rounded-2xl`、半透明背景、柔和阴影。
  - 表单输入高度 52px，圆角 8px，左侧图标，focus 使用主色描边和浅色 ring。
  - 密码显隐使用图标按钮，不使用文字按钮。
  - 安全校验使用独立浅绿色计算块 + 输入框，错误提示使用 amber。
  - 主按钮高度 52px，登录中只显示 loading 图标。
- 主题切换：
  - 登录页使用分段胶囊控制：白天、夜间、跟随系统。
  - 后台内部可继续使用下拉主题菜单。

## 首页/后台入口规范

本项目登录后首页分为普通工作台和管理员入口。管理员会进入 `/app/admin/projects`，因此项目台账页是后台首页的主要设计基准。

### 后台框架

- 使用左侧侧栏 + 顶部紧凑 header + 内容区。
- 侧栏宽度当前为 12rem，折叠宽度 3.25rem。
- Header 高度 56px，左侧显示平台名和业务说明，右侧为全局搜索、通知、用户菜单。
- 内容区使用 `bg-muted/20`，内边距移动端 16px、桌面 20px。
- 侧栏分组：
  - `工作台`：项目管理。
  - `系统`：用户管理、角色管理。
- 侧栏 active 态使用 primary/accent，不使用彩色渐变。

### 管理首页/项目台账

- 顶部概览区使用白底卡片，`rounded-xl`、`border-slate-200`、`shadow-sm`。
- 概览区结构：
  - 左侧：业务胶囊、页面标题、说明。
  - 中间：紧凑统计卡，展示项目总数、在册工人、今日打卡、重点关注。
  - 右侧：主要操作按钮，如 `新增项目`。
- 筛选区：
  - 使用浅底色 `#f8faf9`。
  - 搜索框在左，状态筛选按钮在右或换行。
  - 筛选按钮 active 态为主色实底，inactive 为白底描边。
- 表格：
  - 适合高密度台账，允许 `min-w` 横向滚动，避免压缩重要字段。
  - 表头使用浅底色，行 hover 使用非常浅的绿色底。
  - 主信息放第一列，编号、日期、单位、手机等次信息用 12px 弱文本。
  - 操作列右对齐，使用 `详情`、`编辑`、`删除`，危险操作用红色 ghost 按钮。
- 分页：
  - 底部浅底栏，左侧显示范围，右侧每页数量和上一页/下一页。

### 项目详情页

- 详情页以台账维护为主，不做营销式大标题。
- 顶部保留返回按钮和主要操作：`导出档案`、`编辑项目`/`新增...`。
- 标签导航使用 sticky 浅色条，active 下划线为主色。
- 多模块筛选使用紧凑 grid，字段标签 11px，输入高度 32px。
- 详情表单按业务 section 分组，每组有标题、数量提示、两列字段布局。

## 组件规范

- 基础组件沿用 `shadcn/ui + Radix UI`，不要另起一套组件体系。
- 图标统一使用 `lucide-react`。
- Button：
  - 默认高度 36px，紧凑按钮 32px，登录主按钮 52px。
  - 主操作：主色实底 + 白字。
  - 次操作：outline 或 ghost。
  - 危险操作：destructive 或红色 ghost。
- Card/section：
  - 后台页面卡片圆角优先 `rounded-lg` 或 `rounded-xl`。
  - 只用卡片承载明确的信息单元，不在页面大区块外层套多层卡片。
- Input/Select：
  - 后台常规高度 36px；筛选栏 32px；登录页 52px。
  - focus ring 使用 primary 的透明色，不使用蓝紫默认感。
- Badge：
  - 状态统一使用 outline + 柔和底色。
  - 绿色：正常、在建、有效、在场。
  - amber：筹备、停工、关注、待补图、待完善、未认证。
  - red：预警、异常。
  - slate：竣工、离场。
- Dialog：
  - 表单弹窗可用 `sm:max-w-5xl`。
  - 长表单内部滚动，不让整页滚动失控。
  - Footer 按钮顺序：取消在左，保存/确认在右。
- Toast：
  - 成功：`记录已新增`、`项目已修改`。
  - 失败：明确对象，如 `项目删除失败`。
  - 不要只写 `失败`、`错误`。

## 数据与交互状态

- 列表页必须覆盖 loading、error、empty 三类状态，文案要指向可操作原因：
  - `项目数据加载中`
  - `项目数据加载失败，请检查登录状态或后端服务`
  - `暂无符合条件的项目`
- 表单提交中按钮显示 `保存中...` 或 loading icon，并禁用重复提交。
- 删除类操作必须经过确认弹窗，并说明影响范围。
- 搜索、筛选、分页等控件要保持稳定尺寸，避免结果变化导致布局跳动。
- 后台文案和 mock 通知应使用真实业务场景，不使用英文模板文案。

## 响应式规则

- 登录页：
  - 桌面端展示左侧品牌说明和右侧登录表单。
  - 小屏隐藏左侧说明，表单宽度撑满可用空间。
  - 主题分段控件小屏可隐藏，避免挤压品牌头部。
- 后台：
  - 移动端侧栏通过触发按钮打开。
  - Header 搜索在小屏隐藏。
  - 表格可以横向滚动，不强行挤成不可读列。
  - 过滤器使用 `flex-wrap` 或 grid 换行。

## 暗色模式

- 暗色背景以深青黑和中性黑为主：`#071f22`、`#09272d`、`oklch(0.145 0 0)`。
- primary 暗色使用 `#1aa38e`，不要提高到刺眼的荧光色。
- 边框使用 `white/10` 到 `white/20`，文本用 `white/54`、`white/62`、`muted-foreground` 分层。
- 登录页暗色遮罩要保留工地图片细节，但优先保证登录表单可读性。

## 禁止项与待清理方向

- 禁止继续使用 Quax、Miku、Quick-start、Open Source Starter Template、GitHub star 等模板内容作为业务页面文案。
- 禁止大面积使用 cyan/fuchsia/purple 渐变作为品牌主视觉。
- 禁止在后台管理页面使用营销 hero、夸张卡片堆叠、装饰性光斑和与业务无关的视觉资产。
- 禁止手写 SVG 图标；有图标需求优先用 lucide。
- 后续应将公开落地页改造成山淮业务入口，或直接保持 `/` 跳转登录页，避免用户看到模板首页。

## 新页面检查清单

- 页面是否使用「山淮筑」品牌和工程管理语境？
- 主色是否来自 `#0f6b5d` / `#1aa38e`，而不是旧模板渐变？
- 是否优先使用 shadcn/ui、Radix 和 lucide？
- 后台页面是否紧凑、可扫描、表格和筛选可用？
- loading、error、empty、disabled、delete confirm 是否齐全？
- 深色模式下文字、边框、focus、状态 badge 是否可读？
- 移动端是否没有文字溢出、按钮挤压或表格字段变形？
